module 8 using blocking and nonblocking assignments

  • TL;DR:
    • blocking – combinational, in always body; nonblocking – for sequential ckt, becuz it onlyg assigns when event happens
    • Used in procedural blocks

  • blocking assignment

    • variable = [delay_control] expression OR variable = [event_control] expression
    • eg. a = a + 1;
    • blocks further execution until complete, complete immediately

    • can lead to race condition

      always @(posedge clk)
    a = a + 1;
always @(posedge clk)
    b = a;

    • order matters (p.134)

      1. b and c will not exist in hardware
      always @(posedge clk) begin
    b = a;
    c = b;
    d = c;
end
      1. all exists
      always @(posedge clk) begin
    d = c;
    c = b;
    b = a;
end

  • nonblocking assignment

    • variable <= [delay_control] expression OR variable <= [event_control] expression
    • complete when all executing blocks have blocked

    • for sequential logics!!!

      always @(posedge clk)
    a <= a + 1;
always @(posedge clk)
    b <= a;
      1. procedural blocks unblock on positive edge
      2. execute, and yet update value, just schedule
      3. After executing all triggered blocks to the point where they block, simulator completes the onoblocking assignment by updating values

  • practice

    • use blocking assignment to intermediate(temporary, only be used in one block) variables within sequential procedures

    • assign temporary variables(combinational result) to outputs with nonblocking assignment(sequential ckt)

    • eg

    • eg

      always @(posedge clk) begin
    integer temp;
    temp = a + b;
    q <= temp + c;
end

    • eg. register ?

parameter WIDTH=8;

        data_out <= data_in & load;
